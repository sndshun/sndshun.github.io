    什么是ioc

    ioc即控制反转，也被叫做依赖注入（DI），是对ioc更简单的诠释。

    控制反转是把传统程序代码的控制器交给容器，通过容器实现对象组件的装配和管理。

    什么是aop

    aop面向切面编程是一种程序设计范型，该范型以一种称为切面的语言构造为基础，切面是一种新的模块化机制，用来描述分散在对象、类或方法中的横切关注点。

    利用aop可以对业务逻辑各部分进行隔离，降低耦合度，提高重用性。主要功能是：日志记录，性能统计，安全控制、事物处理、异常处理等等。

    aop可以干什么

    前置通知、后置通知、异常通知、最终通知、环绕通知

    hashmap和hashtable的区别和作用

    HashMap是非线程安全的,HashTable是线程安全的。所以HashMap效率要比HashTable效率高

    HashMap的键和值都允许有null值存在,而HashTable则不行。

    Hashtable是同步的，而HashMap不是。因此，HashMap更适合于单线程环境，而Hashtable适合于多线程环境。

    Java的四种引用，强弱软虚，用到的场景

    强引用：垃圾回收器不会回收

    软引用：如果内存空间足够，垃圾回收器就不会进行回收，如果内存空间不足，垃圾回收器就会进行回收

    弱引用：一旦发现了只有弱引用的对象，垃圾回收器就会进行回收。

    虚引用：如果发现该对象还具有虚引用，就会在回收该对象之前，把这个虚引用加入到与之关联的引用队列中。

    静态变量和实例的区别

    静态变量前要加上 static

    实例变量是某个对象的属性，创建实例对象，实例变量才会分配空间

    静态变量不属于任何实例对象，属于类，也成为类变量，只要加载了类的字节码，不用创建任何实例就能使用。

    Overload和Override的区别

    Overload 重载：同个类可以有多个相同名称的方法，但是参数列表不同，即参数数量，参数类型不同。返回值可以不同，但参数列表一致时不能通过不同返回值实现重载。

    Override 重写：子类方法名称和参数可以跟父类完全一致，调用时会执行子类定义的方法。同时子类方法只能抛出比父类更小的异常，访问权限不能比父类低，返回值也要和被覆盖的方法一致

    String 、StringBuffer与StringBuilder的区别

    String 是内容不可修改的字符串

    StringBuffer 是内容可以修改的字符串

    String 覆盖了equals()和hashcode()，StringBuffer没有，所以StirngBuffer存储到Java集合类会出现问题

    StringBuilder 也表示内容可以修改的字符串，但其线程不安全，运行效率高

    Java面向对象的特征与含义

    封装、继承、多态、抽象

    封装：把对同一事物的操作方法和相关方法放在同一个类中，把方法和操作数据放在同一个类中

    继承： 子类继承父类作为自己的内容，可以加入新的内容或修改父类内容而更合适特殊需求。提供程序重用性和可扩展性

    多态： 是指同一个行为具有多个不同表现形式或形态的能力

    抽象： 找出事物的相似性和共性，将这些事物归为一类，这个类只考虑这些事物的相似和共性，忽略和当前主题不相关的内容

    什么是序列化与反序列化

    序列化就是把Java对象转换为字节流的过程Java反序列化就是指把字节序列恢复为Java对象的过程

    Java对象的序列化与反序列化

    就是将Java转换为字节流的形式再传输，jre本身提供这种支持可以调用OutputStream的writeObject方法来做，被传输的对象必须实现serizlizable接口，javac编译时就会进行特殊处理才可以被writeObject方法操作，这就是所谓的序列化，serizlizable只是标注该对象可被序列化的。

    BIO、NIO、AIO 有什么区别

    BIO是同步阻塞,NIO是同步非阻塞,AIO是异步非阻塞

    1. BIO是阻塞的，NIO是非阻塞的
    2. BIO主要是以I/O流的形式处理数据；NIO以I/O块（buffer）的形式处理数据，效率：块>流
    3. BIO主要是以字节流和字符流操作，NIO是基于channel（通道）、buffer（缓冲区）操作的；selector（选择器）是用来监听channel的
    4. BIO是单向的，要么是输入流要么是输出流，NIO是双向的，可以从channel往buffer读写数据，同时buffer也可以向channel读写数据

    HashMap 和 Hashtable 有什么区别

    这俩采用的算法大概都一样，性能没太大差异。

    HashMap是Hashtable都实现了Map接口，主要区别在于HashMap是轻量级实现，非线程安全的，在只有一个线程访问的情况下，效率要高于Hashtable。

    HashMap允许将null作为key或者value，Hashtable不允许。

    HashMap把Hashtable的contanins方法去掉了，改为containsValue和containKey。因为contanins方法容易让人引起误解。

    Hashtable基本陈旧的Dictionary类，HashMap是Java1.2引进的Map接口的一个实现

    如何决定使用 HashMap 还是 TreeMap

    如果需要得到一个有序的结果时使用TreeMap，但HashMap有更好的性能，所以大多数不需要排序的时候推荐使用HashMap

    迭代器 Iterator 是什么

    迭代器是一种设计模式，它是一个对象，可以遍历并选择序列中的对象，而开发人员不需要了解该序列的底层机构。

    Iterator 怎么使用？有什么特点

    功能比较简单，只能单向移动

    1. 使用方法iterator()要求容器返回一个Iterator。第一次调用Iterator的next()方法时，它返回序列的第
    2. 一个元素。注意：iterator()方法是java.lang.Iterable接口,被Collection继承。
    3. 使用next()获得序列中的下一个元素。
    4. 使用hasNext()检查序列中是否还有元素。
    5. 使用remove()将迭代器新返回的元素删除。
